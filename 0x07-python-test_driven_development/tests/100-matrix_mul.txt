============================
How to Use 100-matrix_mul.py
============================

This module defines a matrix multiplication function ``matrix_mul(m_m, m_n)``.

Usage
=====
``matrix_mul(...)`` returns a new matrix representing the multiplication of
``m_m`` by ``m_n``.

::

    >>> matrix_mul = __import__('100-matrix_mul').matrix_mul
    >>> m_m = [
    ... [1, 2],
    ... [3, 4],
    ... ]
    >>> m_n = m_m
    >>> print(matrix_mul(m_m, m_n))
    [[7, 10], [15, 22]]

::

    >>> m_m = [[1, 2]]
    >>> m_n = [
    ... [3, 4],
    ... [5, 6]
    ... ]
    >>> print(matrix_mul(m_m, m_n))
    [[13, 16]]

The function also works with floating-point numbers.

::

    >>> m_m = [
    ... [1.2, 5.5, 6.2],
    ... [4.66, 12.3, -9.2]
    ... ]
    >>> m_n = [
    ... [5.0, 3.3],
    ... [-2.9, 4.4],
    ... [7.2, 4.4]
    ... ]
    >>> print(matrix_mul(m_m, m_n))
    [[34.69, 55.44000000000001], [-78.61, 29.018000000000008]]

Integers and floats can be combined.

::

    >>> m_m = [
    ... [1, 2.2, 3.3, 4],
    ... [5, 6, 7, 8.8],
    ... ]
    >>> m_n = [
    ... [1.
    1, 2, 3.3],
    ... [4.0, 5.5, 6],
    ... [7, 8, 9],
    ... [10.01, 11, 12.3]
    ... ]
    >>> print(matrix_mul(m_m, m_n))
    [[73.03999999999999, 84.5, 95.4], [166.58800000000002, 195.8, 223.74]]

A minimum of two arguments must be provided. Otherwise, a TypeError is raised.

::

    >>> print(matrix_mul()) # doctest: +NORMALIZE_WHITESPACE

::

    'm_m' and 'm_n'

ValueErrors
===========

If two matrices cannot be multiplied (ie. the row count of ``m_m`` is not
equal to the column count in ``m_n``), a ValueError is raised.

::

    >>> m_m = [
    ... [1, 2],
    ... [3, 4],
    ... ]
    >>> m_n = [
    ... [1, 2],
    ... [2, 3],
    ... [4, 5]
    ... ]
    >>> print(matrix_mul(m_m, m_n))
    Traceback (most recent call last):
    ValueError: m_m and m_n can't be multiplied


The parameters ``m_m`` and ``m_n`` cannot be empty. Otherwise, a ValueError
is raised.

::

    >>> print(matrix_mul([], [[1, 2]]))
    Traceback (most recent call last):
    ValueError: m_m can't be empty

::

    >>> print(matrix_mul([[1, 2]], [[]]))
    Traceback (most recent call last):
    ValueError: m_n can't be empty

::

    >>> print(matrix_mul([[]], []))
    Traceback (most recent call last):
    ValueError: m_m can't be empty

Invalid Matrices
================

The parameters ``m_m`` and ``m_n`` must be lists. If either parameter is
not a list, a TypeError is raised.

::

    >>> print(matrix_mul("not a list", [[1, 2]]))
    Traceback (most recent call last):
    TypeError: m_m must be a list

::

    >>> print(matrix_mul([[1, 2]], "also not a list"))
    Traceback (most recent call last):
    TypeError: m_n must be a list

::

    >>> print(matrix_mul("not a list", "also not a list"))
    Traceback (most recent call last):
    TypeError: m_m must be a list

::

    >>> print(matrix_mul(None, None))
    Traceback (most recent call last):
    TypeError: m_m must be a list

Not just any list - they *must* be lists of lists!

::

    >>> print(matrix_mul([1, 2], [[3, 4]]))
    Traceback (most recent call last):
    TypeError: m_m must be a list of lists

::

    >>> print(matrix_mul([[1, 2]], [3, 4]))

::
